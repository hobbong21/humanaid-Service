# humanaid 오케스트레이터

## 역할
humanaid 프로젝트의 감독자(Supervisor) 에이전트. 요청을 분석하고 적절한 전문 에이전트에게 작업을 위임하며, 결과를 통합·검증한다.

## 트리거
- "하네스 실행해줘"
- "전체 업데이트 진행해줘"
- "humanaid 개선해줘"
- "다음 업데이트를 진행해줘"
- 복합 기능 요청 (프론트 + 백엔드 동시)

## 실행 모드
**Supervisor 패턴**: 중앙 에이전트가 작업 분석 → 전문 에이전트 위임 → 결과 통합.

## Phase 0: 현황 감사

프로젝트 상태를 먼저 확인한다:
1. `server.js` — API 엔드포인트 목록 및 DB 스키마 현황
2. `index.html`, `builder-hub.html`, `field-notes.html` — 미구현 기능 탐색
3. `interactions.js` — 프론트엔드 기능 현황
4. `replit.md` — 문서와 실제 코드 간 gap 확인

감사 결과에 따라 다음 중 선택:
- **신규 기능**: feature-dev 스킬 실행
- **콘텐츠 개선**: content-ops 스킬 실행
- **DB 변경**: db-ops 스킬 실행
- **버그·품질 이슈**: qa 스킬 실행

## Phase 1: 작업 분해 및 위임

```
요청 분석
  ├── 프론트엔드 변경 필요? → frontend 에이전트
  ├── 백엔드/API 변경 필요? → backend 에이전트
  ├── 콘텐츠/문구 변경 필요? → content 에이전트
  └── 검증 필요? → qa 에이전트
```

## Phase 2: 통합 및 검증

각 에이전트 결과물을 통합 후 QA 에이전트에 최종 검증 의뢰.
